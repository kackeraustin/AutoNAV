Overview

AutoNav SearchSets automates the creation of search sets in Navisworks for clash detection workflows. This plugin dramatically reduces the time spent on manual search set creation from hours to minutes.

Key Benefits

  • 98% Time Reduction: Automates manual search set creation
  • Consistent Organization: Standardized folder structure for clash tests
  • Locator Filtering: All searches properly scoped to source files
  • Flexible Customization: Create custom search sets for any property

Installation

  1. Run AutoNavSearchSets_v1.3.1_Setup.exe
  2. Follow the installation wizard
  3. Restart Navisworks Manage 2024
  4. Look for "Auto SearchSets" button in the Add-Ins ribbon tab
System Requirements:
  • Navisworks Manage 2024
  • Windows 10/11 (64-bit)
  • .NET Framework 4.8

Quick Start Guide

Step 1: Open Your Coordination Model

Load all discipline models into Navisworks. Ensure file names contain discipline identifiers:

  • PROJECT_ARCH_L01.nwc
  • PROJECT_MEP_L01.nwc
  • PROJECT_STRC_L01.nwc

Step 2: Create Discipline Search Sets

  1. Click "Auto SearchSets" in Add-Ins ribbon
  2. In Function 1, click "Create Discipline Search Sets"
  3. Result: "1. DISCIPLINES" folder with sets for each discipline

Step 3: Create Element Category Sets

  1. Check which disciplines to process (default: all)
  2. Select parameter to organize by (default: Element Category)
  3. Click "Create Element Search Sets"
  4. Result: "2. CLASH SETS" folder with organized search sets

Step 4 (Optional): Create Custom Sets

  1. Check which disciplines to analyze
  2. Click "ðŸ”„ Analyze Properties"
  3. Select Property Category (e.g., "Element")
  4. Select Property Name (e.g., "Family")
  5. Click "Create Custom Search Sets"
  6. Result: "3. CUSTOM SETS" folder with auto-discovered values

Troubleshooting

Plugin button not appearing

  • Verify installation path: C:\ProgramData\Autodesk\ApplicationPlugins\AutoNAVSearchSets.bundle
  • Restart Navisworks
  • Check Add-Ins ribbon tab (not Home tab)

"No discipline patterns found" error

  • Ensure file names contain discipline identifiers: _ARCH_, _MEP_, _STRC_, etc.
  • Example correct name: PROJECT_ARCH_Level1.nwc

Search sets not scoping correctly

  • Make sure you're using v1.3.1 (check version in dialog title)
  • Run Function 1 first to create discipline sets
  • Verify search sets have locator tag when exported to XML

Performance Tips

  • Function 1: ~5-10 seconds
  • Function 2: ~10-15 seconds per parameter type
  • Function 3: ~30 seconds for comprehensive property discovery
  • Large projects (100+ MB): Allow up to 60 seconds for property scanning
